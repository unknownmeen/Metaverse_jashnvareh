# In the Name of God

## Document of Page Structure and Interactions of the "Building" Platform (Visual Festivals)

### 1. Users Structure and Access Levels
* **Admin:** Has full access to create, edit, and delete streams, select featured images, and submit pinned critique comments.
* **Judge:** Only acts as a viewer, with no management access or special badge (might be needed later for future features).
* **Regular User:** Can participate in streams, upload photos, submit comments, and give star ratings.

### 2. Name Display Logic System
* **Men:** Have only one name (real first and last name). They see the real names of other men and the display (alias) names of women.
* **Women:** Have two names (real and display). They see the real names of all users, but their display name is used for their own profile and posts.

### 3. Top Header Structure
* **Right Side (Title & Navigation):**
  * **View Toggle / Menu Button:** A map icon in a circle next to the title. Clicking it opens a capsule-shaped floating menu containing the navigation items (detailed in section 4).
  * **Site Title:** Displays the site title ("Building" project).
* **Left Side (Tools & Notifications):**
  * **Help Button:** A question mark icon that displays the platform's help page when clicked.
  * **Notifications Button:** A bell icon that opens a drop-down pop-up when clicked.
    * **Notifications Header:** Includes the "Notifications" title and a checked "Read All" button on the left.
    * **Notifications Body:** Displays "There are no notifications" if empty, or a list showing the icon, avatar, text, and time for each notification.

### 4. Main Navigation Bar (Page Access)
1. **Home:** Provides access to the "My Photos" gallery.
2. **Streams:** Provides access to the list of all festivals page.
3. **Profile:** Opens personal settings when clicking the user's avatar.
4. **Admin View:** Admin-only access to enter the stream management dashboard.

### 5. Home Page
* **Empty State:** If the page is empty, an encouraging button invites the user to participate in a stream, redirecting them to the streams page upon clicking.
* **My Photos Section:** Displays a grid view of all images generated by the user so far.
* **Image Interaction:** Clicking on any photo opens its details page (specifications in section 9).

### 6. Streams Page
* **Streams List:** Displays cards related to various festivals.
* **Stream Card Info:** Each card shows the stream cover photo, stream name, and stream status (Opened / Finished).
* **Enter Button:** Clicking on a card enters the "Explore Page" of that specific stream.

### 7. Stream Explore Page (Inside a Festival)
* **Stream Information (Top of page):**
  * Cover and Name.
  * **Concept Description:** Capability to display media (upload/playback video or show photo) along with explanation text.
  * **Form Rules:** Text explaining how it works and the rules of the festival.
* **Create/Upload Image Button:** Active only when the stream status is "Opened" (hidden when "Finished").
* **Featured Images Section:** A gallery of admin-approved photos. Clicking an image opens its details page.
* **Explore All Images:**
  * **Filter Tool:** Capability to filter uploaded images.
  * **Normal Image Gallery:** Clicking a photo opens its details page.

### 8. Admin View (Stream Management Panel)
* **Dashboard (Streams List):**
  * **"+ Create Stream" Button:** A prominent blue button at the top to open the new stream form.
  * **Stream Cards:** Displays the cover, name, generated images count, and status badge.
  * **Actions:** Edit button (opens edit form) and Delete Stream button.
* **Create / Edit Stream Form:**
  1. Stream Name field (Text - Required)
  2. Stream Photo field (Cover upload - Required)
  3. Concept Video section: Option to select and upload a media file (video/photo - Optional).
  4. Concept Description field (Large text box).
  5. Form Rules Description field (Large text box).
  6. Stream Status dropdown: Select between Active/Opened, Finished, Not Opened.
  7. "Save Changes" and "Cancel and Return" buttons.

### 9. Image Details Page
* **Main View:** A main frame displaying the image in a large size.
* **Comments and Ratings:**
  * Star rating submission (Rating).
  * List of user comments.
  * New comment box.
  * *Note:* The comment and rating systems are combined (the user gives a rating and writes their opinion simultaneously).
* **Critique Comment (For Featured Images):** The admin's comment has a distinct style and is pinned above all other comments to act as a formal review.

### 10. User Profile Page
* **Fixed Information:** Phone number and Gender (Female / Male).
* **Editable Information:**
  * Real Full Name.
  * Display Name (This field is only shown in the UI if the gender is set to "Female").
  * Change Password option.

### 11. Login Page
* Mobile number field.
* Password field.
